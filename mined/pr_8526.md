# PR 8526 [MERGED] — ai: add `gh-issue` command to help diagnose GitHub issues
AUTHOR: mitchellh

## BODY
This enables agents (namely Amp) to use `/gh-issue <number/url>` to  begin diagnosing a GitHub issue, explaining the problem, and suggesting a plan of action. This action explicitly prompts the AI to not write code.

**You can run this manually too,** for testing or curiosity or for pasting into another LLM. Execute it like any other script: `.agents/command/gh-issue <issue>`

I've used this manually for months with good results, so now I'm formalizing it in the repo for other contributors.

Example diagnosing #8523:

https://ampcode.com/threads/T-3e26e8cc-83d1-4e3c-9b5e-02d9111909a7

**I'm going to be highly selective about integrating repository-level commands.** I think guiding AI contributors in the right direction is going to result in less AI slop being sent to us. But I want to only add commands that maintainers use and also can vouch for. The best way to vouch is to share something like an Amp thread link that shows it working on real data. 

Ironically, no AI was used to write this PR. I did consult Claude Chat for help on some Nu syntax, but verified it manually with the Nu manual.
